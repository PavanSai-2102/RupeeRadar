# RupeeRadar: Project Context

## Overview
**RupeeRadar** is an AI-powered personal finance assistant designed to help working professionals understand where their money is going by analyzing their bank statement data. 

## The Problem
Bank statements contain a wealth of financial information across UPI, cards, bank transfers, subscriptions, EMIs, and more. However, transaction descriptions are typically messy, inconsistent, and hard to categorize manually, making it difficult for users to gain meaningful insights into their spending habits.

## The Solution
An end-to-end application that allows users to upload raw bank statement data and automatically converts it into meaningful personal finance insights. 

The application answers key questions such as:
- What are the biggest spending categories?
- How much was spent in a given month?
- Which transactions are recurring (subscriptions, EMIs)?
- What was the biggest transaction?
- What are the top behavioral spending insights?

## Core Capabilities
1. **Data Ingestion**: Accept raw bank statement data.
2. **Data Processing & Cleaning**: Extract and clean messy transaction descriptions into a structured format.
3. **Smart Categorization**: Group transactions into meaningful categories (e.g., Food, Travel, Shopping, Bills, EMI, Subscriptions, Salary, Rent, Investments).
4. **Recurring Payment Detection**: Identify recurring transactions (subscriptions, EMIs, rent, SIPs, insurance).
5. **Financial Metrics Calculation**: Calculate total income, total spend, savings, top categories, and biggest transactions.
6. **Insights Generation**: Generate clear, personalized, human-readable insights using actual transaction amounts.
7. **Presentation**: Display the final output via a simple user interface, dashboard, or downloadable report.

## Deliverables & Constraints
- **Final Deliverable**: A deployed or locally runnable application providing a clear personal finance summary.
- **Constraints**: The primary focus is on a working end-to-end prototype rather than perfect support for every possible bank format.
- **Technology Stack**: Open to any technology stack and implementation approach.

## Evaluation Criteria
- Accuracy of data cleaning and categorization.
- Quality and personalization of financial insights.
- Robustness in handling real-world messy transaction descriptions.
- Simplicity and usefulness of the UX/UI.
- Completeness of the end-to-end workflow.
- Privacy-conscious handling of sensitive financial data.
